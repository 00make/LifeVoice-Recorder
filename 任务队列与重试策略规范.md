# 任务队列与重试策略规范（LifeOS）

## 1. 队列分级

- `high`：实时提醒（目标 < 5 分钟）
- `normal`：日常处理
- `low`：归档/低优先级

## 2. 重试策略

- 指数退避：1m → 5m → 15m
- 最大重试：3 次
- 死信队列：进入 `failed` 队列

## 3. 任务幂等

- `task_key = segment_id + pipeline_version`
- 若 task_key 已完成，直接返回结果

## 4. 并发与资源限制

- GPU 任务并发 ≤ 2
- CPU 任务并发 ≤ 4
- 同一 segment_id 不并行
